## Reproducing Figure 7

### Building Token to Expert Mapping Table (TEMT)
TEMT is a mapping table that records, for each dataset's build split, the top-k expert IDs assigned by each scheme for every token. This can be generated by running scripts below in `Libra-Internal`.

```bash
cd Libra-Internal

# Generate TEMT for Qwen3MoE
bash generate_temt_qwen3.sh

# Generate TEMT for GLM-4.5
bash generate_temt_glm45.sh

cd ..
```

### SGLang
Throughput of SGLang can be evaluated by running scripts below in `Libra-Core`. Check `is_ori = True` before running the script.
```bash
cd Libra-Core
git checkout iclr2026_libra

# Qwen3MoE
bash qwen3_throughput_sglang.sh

# GLM-4.5
bash glm45_throughput_sglang.sh

cd ..
```

### EPLB
EPLB requires offline profiling to find hot experts and cold GPUs, and then map hot experts to cold GPUs. Expert placement, which is the result of offline profiling, can be generated by running scripts below in `Libra-Core`. Check `is_ori = True` before running the script.
```bash
cd Libra-Core/eplb
git checkout iclr2026_libra

# Find hot experts and cold GPUs
bash find_hot_experts_and_cold_gpus_qwen3.sh
bash find_hot_experts_and_cold_gpus_glm45.sh

# Create expert placement
bash run_all_cold_gpu_qwen3.sh
bash run_all_cold_gpu_glm45.sh

cd ../..
```

After generating expert placement, throughput of EPLB can be evaluated by running the scripts below in `Libra-Core`.

```bash
cd Libra-Core
# Ensure you are on the iclr2026_libra branch
git checkout iclr2026_libra

# Qwen3MoE
bash qwen3_throughput_eplb.sh

# GLM-4.5
bash glm45_throughput_eplb.sh

cd ..
```

### Lina
Lina requires an expert-selection-path table, which is a lookup table used to predict the next expert to map a token to, given the historical path of previously selected experts. The expert-selection-path table can be generated by running the script below in `Libra-Internal`. Check `is_ori = False` before running the script.

```bash
cd Libra-Internal/lina

# Generate expert-selection-path table
bash run_make_expert_path_table_qwen3.sh
bash run_make_expert_path_table_glm45.sh

cd ../..
```

After generating the expert-selection-path table, throughput of Lina can be evaluated by running the scripts below in `Libra-Core` on the `iclr2026_lina` branch.

```bash
cd Libra-Core
git checkout iclr2026_lina

# Qwen3MoE
bash qwen3_throughput_lina.sh

# GLM-4.5
bash glm45_throughput_lina.sh

cd ..
```

### Libra
By running the scripts below in `Libra-Core`, we can evaluate the throughput of Libra. Check `is_ori = False` before running the script.

```bash
cd Libra-Core
git checkout iclr2026_libra

# Qwen3MoE
bash qwen3_throughput_libra.sh

# GLM-4.5
bash glm45_throughput_libra.sh

cd ..
```

## Reproducing Figure 8

### Throughput
Throughput of SGLang, Lina and Libra is already evaluated in Figure 7.

### Imbalance Ratio
Using TEMT, we can evaluate the imbalance ratio, which is defined as the load of the most burdened GPU divided by the average load across all GPUs. The imbalance ratio of each scheme can be evaluated by running the scripts below in `Libra-Internal`. Hot experts, cold GPUs data, and Lina's expert-selection-path table, which were used in throughput evaluation, are also required to calculate the imbalance ratio.

```bash
# For EPLB
cd Libra-Internal/eplb
# Copy hot experts data generated from Libra-Core
cp ../../Libra-Core/eplb/hot_experts . 

bash evaluate_fluctuation_eplb_2048.sh
bash evaluate_fluctuation_eplb_4096.sh

cd ../..

# For Lina
cd Libra-Internal/lina

bash evaluate_fluctuation_lina_2048.sh
bash evaluate_fluctuation_lina_4096.sh

cd ../..

# For SGLang and Libra
cd Libra-Internal/prefetch_rebalance

bash evaluate_fluctuation_libra_2048.sh
bash evaluate_fluctuation_libra_4096.sh

cd ../..
```

### Combine into Throughput/Imbalance Ratio Fluctuation Data

Throughput fluctuation in the shuffled dataset can be combined by running a python file below in `Libra-Core`. Since Lina and Libra share the `Libra-Core` directory, the throughput result files generated while on the `iclr2026_lina` branch will persist in the directory when you checkout `iclr2026_libra` (assuming the output folder is ignored by git).

```bash
cd Libra-Core
git checkout iclr2026_libra

python get_throughput_fluctuation.py

cd ..
```

Imbalance ratio fluctuation in the shuffled dataset can be combined by running a python file below in `Libra-Internal`.

```bash
cd Libra-Internal
python get_imbalance_ratio_fluctuation.py
cd ..
```
